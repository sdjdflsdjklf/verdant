# Obsidian Garden 产品路线图

> 最后更新: 2026-05-20 — Phase 2 部分完成（全文搜索 + Pro/Free 区分）
> 本文档定义长期产品规划，包括免费/Pro 功能划分和版本演进路线。

---

## 一、核心理念

**Obsidian Garden = 一键发布你的 Obsidian 笔记为独立博客**

- 零后端、零月费、零技术门槛
- 免费用户获得完整的博客体验
- Pro 用户获得让博客被发现、被喜欢的进阶能力

---

## 二、功能矩阵

### 2.1 Free + Pro（所有人都有）

这部分是博客的基础能力，免费版就应该完整提供。Pro 不做阉割。

| 功能 | 说明 | 优先级 |
|---|---|---|
| **一键部署到 GitHub Pages** | 核心卖点。用户提供 GitHub 仓库，插件自动推送到 Pages。支持新建仓库和已有仓库。 | P0 |
| **自动目录（TOC）** | 每篇文章自动生成侧边/顶部目录，支持多级标题。 | P0 |
| **关系图谱导航** | 利用 Obsidian 的 `[[链接]]` 关系，在文章底部展示相关笔记。左侧「上一篇」右侧「下一篇」按链接相关度排序。 | P0 |
| **评论系统** | 集成 giscus（基于 GitHub Discussions），零成本。读者用 GitHub 账号即可评论。 | P0 |
| **RSS 订阅** | 自动生成 `/feed.xml`，读者可订阅博客更新。 | P1 |

### 2.2 Pro 专属（付费 $19）

Pro 卖点围绕一个核心主题：**「让认真写博客的人得到更多」**——更多读者、更好看、更好找。

| 功能 | 一句话卖点 | 优先级 |
|---|---|---|
| **全文搜索 🔍** | 笔记多了找不到？Pro 全文索引所有内容，支持模糊匹配 + 标签过滤。免费版只搜标题。 | P0 |
| **AI 样式助手 🤖** | 不懂 CSS？在发布预览右侧的 AI 对话中描述你想要的样式（"标题用深蓝色"、"卡片圆角大一点"），AI 直接生成 CSS，预览实时更新。 | P0 |
| **流量统计 📊** | 你的博客到底有没有人看？哪些文章最受欢迎？从哪搜到的？插件内嵌轻量面板，一目了然。 | P1 |
| **SEO 增强 🌐** | 自定义 meta 描述、自动生成 OG 图（发朋友圈/推特好看）、sitemap。让 Google 能找到你的文章。 | P1 |
| **图片优化 🖼️** | 笔记中的图片自动转 WebP + 压缩，加快页面加载速度。 | P2 |

### 2.3 不会做的（明确不纳入）

| 功能 | 原因 |
|---|---|
| 图谱可视化（关系图嵌入网页） | 博客读者不需要，图个好看但没有实际用途 |
| AI 摘要/自动标签 | 依赖外部模型，效果不可控，引入 API 成本 |
| 定时发布 | 场景太窄，绝大多数用户不需要 |
| 自定义 CSS（手动编辑） | 用户直接改 GitHub repo 文件即可 |

## 三、版本与定价

### 定价策略

| 版本 | 价格 | 目标用户 |
|---|---|---|
| Free | ¥0 / $0 | 所有 Obsidian 用户，需要把笔记发布为博客 |
| Pro | $19 一次性 / ¥99 一次性（面包多） | 认真写博客、希望被更多读者发现的人 |

### 为什么是 $19 一次性？

- 零后端意味着零持续成本，不需要月费
- 目标用户是 Obsidian 用户群，习惯一次性付费
- 对标同类产品（Obsidian Publish $4/月 起），$19 一次性有极强竞争力

### Pro 权益一句话

**「你可以免费地发布博客。但想让别人能找到你的博客、知道哪些文章最受欢迎、一键换皮肤——就是 Pro。」**

## 五、技术约束（不变）

| 项目 | 规则 |
|---|---|
| 后端 | 零。没有服务器，没有数据库，没有 OAuth 回调 |
| 托管 | 仅 GitHub Pages（MVP），Netlify / Vercel 暂不纳入 |
| 构建 | esbuild |
| 许可证验证 | RSA-2048 离线验证，无水印，无电话回家 |
| 本地化 | 插件 UI 英文，文档/产品页面中英双语 |
| AI 接口 | 用户自带 API key（OpenAI / DeepSeek），插件无内置 API 成本 |

## 六、常见问题

### Q: 免费版限制篇数吗？
**A:** 限制 10 篇。但 Pro 的核心卖点是搜索 + AI 样式 + 统计，而不是单纯的解锁笔记数量。「因为笔记太多所以付费」不是一个好故事，所以 Pro 还有更多能力。

### Q: 图床怎么办？
**A:** 图片直接上传到 GitHub 仓库，走 GitHub raw CDN。无需额外配置。

### Q: 会不会跑路？
**A:** 零后端=零大厂依赖。即使停止维护，你能一直用。所有代码开源。
